UI optimisation and new patterns

Inventory (Book of records)

Problem

The current DigiCert One design system was quite complicated. The key products felt antiquated and lacked modern-day practices. Based on previous technical difficulties and challenges with component scalability, a need for a whole new current style guide for the DigiCert One product was discovered.

Goal
  • Keep the application consistent throughout.
  • Create a scalable design that allows the user to manage both our single customer and large corporations.
  • Make the most of your screen real estate by avoiding elements that aren't directly related to the user's work.

Solution
  • Improvements to the page layout.
  • Added the advanced rule capability and updated the workflow for filters.
  • Added a date filter by default so that the user can easily understand the basis on which the data is filtered, such as Today, the previous week, the previous month, the previous quarter, the previous six months, the previous twelve months, and a custom date.
  • Added graph portlets to give users a view of a small dashboard that they may tailor to their needs.
  • Added the notion of views so that users could create personalised screens that suited their needs and were not forced to see unnecessary information every time they logged in.
  • Additional local vs. global action items have been added to the right side of the page.


Project duration: 2 weeks

Test phase

Usability testing

Following the completion of the design, we began testing the prototype. The test was conducted via Zoom video calls, and the participants were given the following tasks while I observed how they navigated through the page.


Tasks performed
  • Find out what user think of the "primary" inventory view.
  • Get input on the "richer" inventory visualisation view (top level feedback - not much drilling down).
  • Integrate data: For example, integrate private certificates from Microsoft CA.
  • How would you filter Microsoft CA (Certificate authority) and DigiCert Private SSL SHA1 CA -G2?
  • How do you save this view now that you've added your first filter to the table so that the next time you visit this page, you'll see your default values?
  • Filter for certificates that will expire within 30 days.
  • Perform the bulk action of renewing those certificates that are about to expire.


Findings from the user testing

  • They didn't notice the save view option till they clicked on manage view.
  • Because there were more clicks, it was not possible to apply filters in bulk.
  • The user became perplexed by the terminology of views and my saved filter.

Final suggestions and feedback implementation

Following the completion of the testing and comprehension of the pain points. We revised our designs in response to client input. Here are the modifications we made to the filter workflow and manage view.

  • When there is a change in the page, a green colour bubble icon is added to draw the user's attention.
  • Prior to updating the quick filter design, there was no way to tell which filter was in the selected mode.
  • To avoid misunderstanding, the language was changed from My saved filters to My saved view.
  • Users can now see counter value as how many sub-filters are¬†within the filter they have selected.
  • The goal was to provide a quick view to update the applied filter so that the user does not have to go to the side rail every time to edit one single filter.
