Certificate lifecycle

Requesting and enrolling for a certificate

Problem

Profile was a key features of Enterprise PKI. Users that want to request a certificate build profiles, which are nothing more than certificate templates. There are various types of certificates:

  • User certificates: Certificates that allow users to perform actions that would otherwise be prohibited.
  • Device certificates: An electronic document that is permanently integrated into a piece of hardware.
  • Server certificates: Are typically given to hostnames, which could be a machine name (such as "XYZ-SERVER-01") or a domain name (such as "www.example.com").

The current process was extremely difficult. Some of the current issues are listed below:

  • It wasn't clear how to add the profile.
  • Confusion arose over the enrollment code because it wasn't clear whether length or code was necessary.
  • Users had trouble adding a new field to "Certificate fields" because there was no dropdown indicator.
  • Primary Uses Confusion resulted from True/False setup.
  • Toggle to modify administrative contacts can be a little confusing.
  • Side rail being off to the right might make it easy to miss.
  • Jargon that would benefit from more details.
  • Difficult to return to certain steps.

Goal
  • Enhance the interface.
  • Simplify the steps.
  • To reduce clicks and speed up decision-making.
  • Present more relevant details up front.

Solution
  • Improvements to the page layout.
  • Presented a wizard method.
  • Lowered the number of stages from 10-15 to a maximum of 5.
  • The stages should have appropriate labelling that makes it obvious what each step involves.
  • Wherever it was needed, information text was included to help the user with the task.


Project duration: 3 weeks

Key findings from the user research
  • Make it clearer that selecting templates initiates enrollment process.
  • Shrink size of text box to match number of characters needed for enrollment code.
  • Keep drop down signifiers consistent.
  • Improve True/False configuration.
  • Change edit toggle to edit icon.
  • Bring side rail to the top.
  • Make sure potential jargon has a way to get additional information.

Test phase

Usability testing

I tested the prototype with users after the visual design was finished to check how user-friendly the profile workflow is. Participants in the test were given the following tasks as I watched how they used the application throughout a series of zoom calls with various people.


Tasks performed
  • How can a profile be created?
  • What exactly do you understand by renewal flows?
  • How are the Subject domain name and Subject Alternate Name configured?
  • Where would you go if you wanted to change your admin contact information?
  • How do you add fields for authentication enrollment?


Findings from the user testing

  • Users were easily able to understand the contextual button provided on the profile page as to how they can create a profile from a template.
  • With the help of info text users where able to understand what renewal flows was all about.
  • Users were happy about the tooltips we provided to give a summary of what each step contains.
  • Users were able to configure the Subject domain name and Subject Alternate Name.

Final conclusion

According to our analysis, the profile workflow scored 87 on the Usability Scale, which is great! Based on customer feedback and internal plan goals, we have been continuously upgrading since the launch in 2020.
